1. Ideia do Projeto

O projeto tem como objetivo desenvolver uma solução tecnológica para auxiliar na organização de abrigos em situações de enchentes no Brasil. A proposta é facilitar o acesso à informação, conectando pessoas que precisam de ajuda com aquelas que podem ajudar.

======================

2. Problema Escolhido

O problema escolhido foi a falta de informação organizada sobre abrigos. Em momentos de enchente, muitas pessoas não sabem quais locais ainda possuem vagas disponíveis, e as doações acabam sendo mal distribuídas, causando excesso em alguns abrigos e falta em outros.

====================

3. Solução Proposta

A solução desenvolvida consiste em um sistema onde os responsáveis pelos abrigos podem atualizar informações importantes, como:

- quantidade de vagas disponíveis  
- status da comida (suficiente, baixo ou em falta)  
- status da água  

Dessa forma, é possível direcionar melhor tanto as pessoas quanto as doações, evitando deslocamentos desnecessários e melhorando a distribuição de recursos.

====================
4. Estrutura da Solução

O sistema foi desenvolvido utilizando uma arquitetura simples, composta por:

- Backend: desenvolvido em Node.js com Express, responsável por gerenciar as rotas e requisições  
- Banco de Dados: SQLite, utilizado para armazenar as informações dos abrigos  
- Frontend: interface em HTML, CSS e JavaScript, responsável por exibir os dados de forma clara para o usuário  

A aplicação permite:

- cadastrar novos abrigos  
- visualizar a lista de abrigos  
- acompanhar o status de recursos (comida, água e vagas)  

======================
5. Considerações Finais

O projeto foi desenvolvido com foco em simplicidade e funcionalidade, buscando resolver um problema real de forma prática. Mesmo sendo uma solução básica, ela contribui para uma melhor organização em situações de emergência, onde o acesso rápido à informação pode fazer diferença.
